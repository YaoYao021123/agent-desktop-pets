Agent Desktop PetAgent Desktop Pet

把 Codex 和其他 coding agent 的运行状态放到桌面小屏上。StickS3 通过本地 BLE bridge 接收 agent 状态、额度、重置倒计时和权限请求；你看一眼小屏就知道现在是 busy、idle、attention，或者额度是否接近危险区。

当前定位：不是营销页，也不是复杂仪表盘；它是一个常亮在桌面的 agent companion。
设计原则：小屏只显示高价值状态，长说明留在本页右侧滚动文档。
01 玩法glance-first

默认只需要看屏幕和按两颗键

设备平时显示宠物、5h 剩余额度、7d 剩余额度、reset countdown 和 token 数。Codex 运行时进入 busy；需要权限时进入 attention；任务完成、摇晃和按键互动会触发对应宠物状态。

sleep idle busy attention completed dizzy heart
  • 看额度5h 和 7d 是两个独立窗口；屏幕显示 remaining quota，桥接 JSON 里发送的是 used percentage。
  • 看状态LIVE 表示本地 bridge 正在连接；WAIT 表示设备还没收到有效数据或 BLE 未连接。
  • 做决定权限请求出现时，KEY1 accept，KEY2 cancel；如果设备离线，Codex 仍保留电脑端原生确认。
02 能力local bridge

现在已经打通的能力

  • BLE 本地桥StickS3 广播名类似 Codex-XXXX，电脑上的 Python bridge 发送紧凑 JSON，不需要 Wi-Fi。
  • 额度仪表显示 5 小时与 7 天剩余额度、两个窗口的 reset countdown，以及累计 tokens。
  • 权限响应Codex PermissionRequest hook 会推送到设备；设备通过 BLE 把 accept/cancel 发回电脑。
  • 宠物资源支持 LittleFS character pack 和内置 ASCII species；可以在设置里切换。
  • 主题识别桥接 JSON 支持 agent 字段；当前已为 Codex、Claude、Generic 准备主题映射。

03 连接Codex-XXXX

从刷机到 LIVE 的路径

刷入 firmware 和 LittleFS使用 PlatformIO 运行 build、upload、uploadfs；刷机时按住 lower-left side button 进入 flashing mode。
安装 Codex plugin安装 bleak，在 Codex 插件市场加入本 repo，选择 Codex Usage Stick Local。
启用并信任 hooks生成 SessionStart、UserPromptSubmit、PermissionRequest 三个 hook，并在 Codex 弹窗中确认信任。
触发 BLE pairing打开 Mac 蓝牙，发送任意 Codex prompt；设备显示配对码后在电脑端完成配对。
确认 LIVE屏幕出现 CODEX QUOTA / LIVE、额度条和 tokens，即表示链路正常。
# firmware
pio run -e m5stack-sticks3
pio run -e m5stack-sticks3 -t upload
pio run -e m5stack-sticks3 -t uploadfs

# first BLE pairing fallback
python3 ~/.codex/plugins/cache/codex-usage-stick-marketplace/codex-usage-stick/0.4.0/scripts/codex_usage_ble_bridge.py --verbose --state busy
04 按键A next / B confirm

KEY1、KEY2、PWR 的实际语义

按键会根据界面上下文变化。日常记忆方式是：KEY1 负责下一项或切视图，KEY2 负责确认或互动，PWR 负责熄屏和唤醒。

场景 KEY1 短按 KEY1 长按 KEY2 短按 PWR
仪表 切换 quota / info 页面 打开或关闭菜单 heart 互动 熄屏 / 唤醒
权限 accept 非主要确认动作 cancel 熄屏 / 唤醒
菜单 下一项 关闭菜单 确认当前项 熄屏 / 唤醒
设置 下一项 退出设置 修改当前设置 熄屏 / 唤醒
info / pet 切回主视图 打开菜单 翻页或切宠物页 熄屏 / 唤醒
05 关机safe power

熄屏、关机、换电脑要分开处理

  • 熄屏PWR 短按只让 LCD sleep；再按 PWR 或按 A/B 会唤醒。唤醒用的那次 A/B 会被固件吞掉，避免误开菜单。
  • 关机长按 KEY1 打开菜单，选到 turn off，KEY2 确认；也可以使用 StickS3 的硬件长按电源关机逻辑。
  • 换电脑先停止旧电脑的 bridge 或退出 Codex，再在新电脑提交 prompt。一个 StickS3 同时只建议连一台电脑。
# stop bridge before moving to another computer
python3 plugins/codex-usage-stick/scripts/start_bridge.py --stop

# inspect logs
tail -n 20 ~/.codex/codex-usage-bridge/hook.log
tail -n 40 ~/.codex/codex-usage-bridge/bridge.log
06 拓展roadmap

后续扩展方向

小屏继续保持克制：状态、确认、额度、少量情绪反馈。复杂解释和历史数据放在电脑端或文档页。

  • 多 agent 主题：Codex、Claude、Generic、自定义 agent 自动换色。
  • 更细颗粒度权限面板：展示路径、命令类型、风险等级，但不牺牲一键确认。
  • 宠物生成流水线：把 app pet、自定义 GIF、hatch-pet 统一转为 LittleFS character pack。
  • 工作节奏统计：响应速度、批准/拒绝次数、token level、nap time。
  • 更多交互：提醒、小成就、情绪反馈，但不把小屏做成复杂操作台。
