01 What It Does

把 AI 工作流放到桌面小屏上

StickS3 不替代 Mac 大屏，而是承担“常亮提示”和“快速确认”：你抬眼就能看见 agent 是否需要你，伸手就能处理审批。

03 Hardware Buttons

两颗实体键完成主要操作

页面上方按钮模拟真实硬件效果。日常使用时，A 用来继续或批准，B 用来看详情或拒绝；B 长按进入 Fn 语音输入。长时间无操作会进入 Pet idle，B 双击可关闭自动待机。

日常浏览

  • A下一页；Agent Hub 中先翻 agent 分组
  • B查看当前页面详情
  • B x2从详情返回；普通页回到 Home
  • A hold快速查看设备状态
  • B holdFn 语音输入，打开 Codex dictation
  • Idle无操作进入 Pet；A / B 唤醒，B x2 关闭自动待机

审批请求

  • Prompt审批请求会覆盖当前屏幕
  • A允许 agent 继续执行
  • B拒绝这次操作
  • B x2快速取消并回到安全状态

状态映射

Mac desktop透明浮窗常驻桌面，支持拖拽、点击、自动移动和系统托盘管理。 Codex hooksPreToolUse、PostToolUse、PermissionRequest、Stop 映射到 pet state。 StickS3只显示当前角色、agent 情绪和 A/B 操作，不承载完整宠物管理器。 Codex package官方 pet 包保留轻量 pet.json；atlas 和 state 可按默认 8x9 定义解析。 Sprite atlas1536x1872、8x9、单帧 192x208，9 个状态行。 Idle layer无操作进入 waiting 语义的 Pet idle；审批覆盖层永远优先。 Review stateEdit / Write / code review 类事件可以进入 review，比普通 busy 更明确。
06 User Scenarios

它适合放在真实桌面工作流里

这个页面展示的是产品体验，不是协议调试台。核心目标是让你少切窗口、少错过审批、少猜 agent 状态。
编码时你可以一眼看到 Codex 是否还在跑、当前任务是否完成，以及是否进入异常状态。
等待审批时权限请求直接出现在硬件屏幕上，A 批准，B 拒绝，不需要先找到 Mac 上的弹窗。
语音输入时长按 B 作为 Fn 快捷键，直接打开 Codex 语音输入，把临时想法说给 agent。
专注工作时Schedule 显示下一段任务；无操作后进入 Pet idle，用更柔和的方式展示 agent 状态。
排查问题时Power 页面只保留 CPU、RAM 和连接状态，快速判断是 agent、Mac 负载还是连接出了问题。
